That error is classic enterprise TLS interception / missing corporate root CA on the node(s). Your host can reach GitHub, but Python/Ansible cannot validate the SSL chain:

CERTIFICATE_VERIFY_FAILED: unable to get issuer certificate


Option B (recommended for production): install your corporate CA so validation works

This is the “right” way if you’re going to keep using Ansible and pulling from HTTPS repos.

What you need

Ask your network/security team for the root CA certificate used by your proxy/SSL inspection (often a .crt or .pem).

Then install it on all nodes

Once you have the CA file (say corp-root-ca.crt), we can add a role task like:

- name: Install corporate root CA
  ansible.builtin.copy:
    src: corp-root-ca.crt
    dest: /etc/pki/ca-trust/source/anchors/corp-root-ca.crt
    owner: root
    group: root
    mode: "0644"

- name: Update CA trust
  ansible.builtin.command: update-ca-trust
  changed_when: true


After that, get_url with validate_certs: true will work.

✅ This also helps curl, dnf, and other tools trust outbound TLS properly.
